# LS-TENANT-005 — Tenant Authorization Simulator (Integrated + Predictive Access)

## Objective

Implement a **fully integrated Tenant Authorization Simulator** that allows tenant admins to:

* simulate access decisions for users
* test “what-if” scenarios (context + assignments)
* predict outcomes before making changes
* understand policy impacts in real time

This completes the tenant authorization loop:

```text
See → Understand → Predict → Act
```

This feature must build on **LS-COR-AUT-011D (simulation engine)** and integrate it seamlessly into the tenant UI.

## Architectural Principles

### Reuse Simulation Engine (No Duplication)

* MUST use:

  ```text
  POST /api/admin/authorization/simulate
  ```
* no frontend logic for authorization decisions

---

### Tenant-Scoped Simulation

* tenantId derived from session
* no cross-tenant simulation allowed

### Predictive UX (Core Goal)

* simulate before making changes
* enable “what happens if…” scenarios

### Explainability Parity

# 7. Simulation Result UI (CRITICAL)

---

## A. Decision Header

Display:

* ✅ ALLOW (green)
* ❌ DENY (red)

---

## B. Summary

* permission present (yes/no)
* role fallback used
* reason text

---

## C. Access Path

```text
User → Group → Role → Permission
```

---

## D. Policy Evaluation

For each policy:

* policy name
* effect (Allow/Deny)
* priority
* result

---

### Rule Breakdown

Show:

* field
* operator
* expected vs actual
* pass/fail

---

## Example

```text
Policy: approval.limit
Rule: amount < 50000
Actual: 75000 → FAILED
Result: DENY
```

---

## E. Deny Override Indicator

Show clearly:

```text
Denied due to policy override
```
