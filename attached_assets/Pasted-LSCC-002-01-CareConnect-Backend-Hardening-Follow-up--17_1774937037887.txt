--------------------------------------------------
A. PROVIDER ORG BACKFILL TOOLING
--------------------------------------------------

Provide a safe and deterministic way to identify and backfill providers missing OrganizationId.

Minimum requirements:

1. Ability to identify providers with:
   Provider.OrganizationId == null

2. Provide an admin-only mechanism to link providers:
   Acceptable:
   - bulk endpoint
   - batch command/service
   - admin tool endpoint

3. Must NOT:
   - auto-guess organization mappings
   - assign orgs without explicit input or deterministic rule

4. Must be idempotent:
   - re-running must not corrupt data

5. Must report:
   - total processed
   - updated count
   - skipped count
   - unresolved count

Fallback behavior:
- unresolved providers remain unlinked
- must NOT expand visibility

--------------------------------------------------
B. LEGACY APPOINTMENT ORG-ID BACKFILL
--------------------------------------------------

Populate missing org IDs for existing appointments.

Target fields:
- ReferringOrganizationId
- ReceivingOrganizationId

Requirements:

1. Identify appointments where org IDs are null

2. Derive values ONLY if:
   - appointment has valid referral
   - referral has valid org relationship

3. Update appointment only when both org IDs can be safely determined

4. Leave unresolved records unchanged

5. Must be idempotent

6. Must report:
   - updated count
   - skipped count
   - unresolved count

Fallback behavior:
- unresolved records remain admin-visible only
- do NOT infer or guess
