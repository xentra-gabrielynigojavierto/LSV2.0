# REQUIRED IMPLEMENTATION

# A. Formatting Service

Create:

`ReportFormattingService`

## Responsibilities
- accept:
  - raw rows (post-query, post-formula)
  - formatting config
- return:
  - formatted result set

## Design
- stateless
- reusable
- no DB access
- no external dependencies

---

# B. Formatting Config Mapping

Use existing:
- `FormattingConfigJson`

Parse into:
- column → formatting rule

Each rule should include:
- column key
- format type
- options

---

# C. Supported Formatting Types

Implement the following:

---

## 1. Currency
- apply:
  - prefix (e.g., "$")
  - decimal places
- include thousand separators

Example:
1234.5 → $1,234.50

---

## 2. Number
- decimal precision
- thousand separators

---

## 3. Percentage
- optional multiply by 100 (decide once, document)
- append `%`
- apply precision

---

## 4. Date / DateTime
- apply format string
- examples:
  - yyyy-MM-dd
  - MM/dd/yyyy

---

## 5. Boolean
- map:
  - true → custom label (e.g., "Yes")
  - false → custom label (e.g., "No")

---

## 6. Text
- pass-through
- optional fallback for nulls

---
