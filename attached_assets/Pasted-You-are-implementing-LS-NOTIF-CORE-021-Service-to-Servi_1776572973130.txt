================================================================================
OBJECTIVE
================================================================================

Eliminate anonymous submission to the Notifications service and implement a
secure service-to-service authentication model.

Replace reliance on:
   AllowAnonymous + X-Tenant-Id

with:
   JWT-based authenticated service identity.

================================================================================
IN SCOPE
================================================================================

1. Require authentication for POST /v1/notifications

2. Implement service authentication:
   - validate JWT
   - extract claims:
       tenant_id
       service_name
       roles
       sub (identity)

3. Add ServiceOnly authorization policy

4. Enforce tenant from claims (NOT headers)

5. Maintain temporary backward compatibility:
   - allow header-based fallback
   - log all legacy usage

6. Update producer integrations:
   - Liens
   - Comms
   - Reports
   - Flow

================================================================================
OUT OF SCOPE
================================================================================

- No changes to sending logic
- No UI changes
- No template changes
- No provider changes

================================================================================
IMPLEMENTATION REQUIREMENTS
================================================================================

1. POST endpoint:
   - Replace AllowAnonymous with RequireAuthorization
   - Accept:
       user tokens OR service tokens

2. Claims extraction:
   - tenant_id
   - service_name
   - roles

3. Tenant enforcement:
   - use claim-based tenant
   - ignore header if authenticated
   - validate mismatch

4. Service identity:
   - require service_name claim
   - validate role

5. Backward compatibility:
   - if no JWT:
       allow request ONLY if header present
       log warning "LEGACY SUBMISSION"
   - prepare for removal later

6. Audit:
   - include service_name
   - include tenant_id
   - include correlationId
