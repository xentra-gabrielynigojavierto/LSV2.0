# LS-ID-TNT-017-004 — Audit UI Viewer (Control Center)

## FEATURE OBJECTIVE

Implement a Control Center audit viewer so platform administrators can search, filter, inspect, and review the permission-change audit events already being emitted by the Identity service audit pipeline.

This feature turns the existing backend audit foundation into an operational governance tool by exposing:
- audit event list/search
- tenant-scoped filtering
- actor / target / event-type filtering
- timestamp-based review
- before/after diff inspection
- clear event detail drill-down

This is a **UI + query integration feature**. It does NOT redesign the audit ingestion pipeline.

---

## WHY THIS FEATURE COMES NEXT

Current state:
- LS-ID-TNT-017-001 confirmed that permission-change audit capture is already in place and now complete
- all permission mutations emit structured events
- before/after payloads exist
- tenant isolation exists
- fail-safe centralized audit publishing exists

What is missing:
- no operational UI to inspect those events
- no easy way for admins/support/compliance to answer:
  - who changed access?
  - when did it change?
  - what exactly changed?
  - which tenant/user/group was affected?

This feature makes the audit system usable by humans.
