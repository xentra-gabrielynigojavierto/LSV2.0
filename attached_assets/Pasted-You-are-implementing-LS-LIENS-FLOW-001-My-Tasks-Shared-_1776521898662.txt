==================================================
OBJECTIVE
==================================================

Implement a lightweight operational task manager called My Tasks inside Synq Liens, plus a shared workflow configuration model editable from both:

1. Tenant-side:
   Synq Liens → Product Settings → Workflow

2. Admin-side:
   Control Center → Tenant → Synq Liens → Workflow

This is NOT a full workflow engine.

Tasks must remain operationally useful and loosely coupled to workflow.

==================================================
MANDATORY ARCHITECTURE RULES
==================================================

Use v2 shared services only. Do not use v1 equivalents.

Mandatory shared services:
- Identity
- Audit
- Notifications
- Documents only if needed later; do not overbuild attachments now

Preserve Synq Liens donor behavior and API style where practical while implementing on v2 architecture.

Do not over-engineer this feature.

This is Phase 1 only:
- no BPM engine
- no dependency engine
- no auto-generated workflow tasks
- no SLA enforcement logic
- no workflow runtime orchestration
- no automatic stage progression from task completion

==================================================
HIGH-LEVEL FEATURE SUMMARY
==================================================

Build two connected but distinct capabilities:

A. My Tasks
- lightweight operational task manager
- task creation and assignment
- status and priority tracking
- linked to case and/or one or more liens
- visible from case/lien contexts
- linked task activity flows into Case Events / Timeline

B. Shared Workflow Configuration
- workflow definition for Synq Liens
- editable by tenant admins from Synq Liens Product Settings
- editable by platform admins from Control Center
- both surfaces manage the SAME underlying workflow configuration
- all changes are audited
- UI shows Last Updated, Updated By, Update Source, Version

==================================================
PHASE 1 PRODUCT PRINCIPLES
==================================================

1. Tasks are independent from workflow in Phase 1
2. Workflow defines structure, not enforcement
3. Workflow configuration is single-source-of-truth
4. Tenant admin and platform admin can both manage workflow
5. All significant changes are auditable
6. Case timeline must reflect linked task activity
7. Build for future Flow alignment without forcing it now

STEP 2 — IMPLEMENT MY TASKS

- lienId

If multiple liens are linked, they should belong to the same case context where applicable.

Add sensible validation to prevent ambiguous cross-case multi-lien grouping.

Preferred behavior:
- If task created from Case page, auto-populate case
- If task created from Lien page, auto-populate lien and parent case if available
