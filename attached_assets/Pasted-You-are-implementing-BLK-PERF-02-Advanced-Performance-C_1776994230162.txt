PART A — IDENTIFY CACHEABLE READ PATHS

--------------------------------------------------

Audit endpoints for safe caching candidates.

Prioritize:

1. Public endpoints
   - `/api/public/network/*`
   - provider lists / markers
   - network summaries

2. Tenant read endpoints
   - static reference data (network config, provider metadata)
   - low-frequency-changing data

3. Admin dashboards (bounded caching)
   - summary metrics (short TTL)

DO NOT CACHE:

- user-specific data
- authorization-sensitive results
- write-heavy or rapidly mutating data
- anything dependent on request identity unless cache key includes it

For each candidate, document:
- cache key shape
- tenant dependency
- expected TTL
- invalidation considerations

--------------------------------------------------

PART B — IMPLEMENT TENANT-AWARE IN-MEMORY CACHING

--------------------------------------------------

Use IMemoryCache (or existing abstraction if present).

Rules:

1. Cache key MUST include tenant context:
   Example:
   - `network:list:{tenantId}`
   - `provider:markers:{tenantId}:{networkId}`

2. Cache must NOT mix tenants under any condition.

3. Cache entries must include:
   - absolute expiration (TTL)
   - optional size/priority if supported

4. Suggested TTLs:
   - public network data: 30–120 seconds
   - admin summaries: 10–30 seconds
   - provider metadata: 60–180 seconds

5. Use helper methods if needed:
   - GetOrCreateAsync pattern
   - small wrapper around IMemoryCache for consistency

6. Ensure cache population happens AFTER tenant validation.

--------------------------------------------------

PART C — SAFE CACHE INVALIDATION STRATEGY

--------------------------------------------------

You are NOT building a full invalidation system.

Instead:

- rely on short TTLs
- invalidate manually where trivial and safe

Optional (only if trivial):
- after write operations (e.g. provider activation), remove affected cache keys

DO NOT:
- introduce complex pub/sub invalidation
- attempt cross-service invalidation

Document limitations clearly.
