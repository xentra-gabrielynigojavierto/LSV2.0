2. BRANDING CREATE FLOW
==================================================

Add tenant self-service create flow.

Expected backend endpoint:
- POST /notifications/v1/branding

The tenant portal must NOT ask the user to manually type tenantId.
Tenant identity must come from authenticated tenant context.

Create fields at minimum:
- productType
- brandName
- logoUrl
- primaryColor
- secondaryColor
- accentColor
- textColor
- backgroundColor
- buttonRadius
- fontFamily
- emailHeaderHtml
- emailFooterHtml
- supportEmail
- supportPhone
- websiteUrl

Validation requirements:
- required fields:
  - productType
  - brandName
- basic validation where practical:
  - email format
  - URL format
  - color format
- backend conflict errors surfaced clearly
- prevent duplicate branding for the same tenant + productType where backend rejects it

Behavior:
- on success:
  - revalidate branding data
  - return to list or stay in-place with clear success state

Do NOT add file upload infrastructure unless platform asset handling already exists and is trivial to reuse.
Logo URL input is acceptable for this phase.

- show success/error states clearly
- revalidate branding data after update

Do NOT expose tenantId editing.
