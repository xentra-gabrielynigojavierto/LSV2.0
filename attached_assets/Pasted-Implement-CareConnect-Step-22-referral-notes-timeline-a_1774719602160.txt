Implement CareConnect Step 22: referral notes timeline + appointment notes + attachments integration prep.

Goal:
Make CareConnect operationally useful for coordinators by adding:
1. referral notes timeline
2. appointment notes
3. attachment metadata model and API prep for future SynqDocs integration

Context:
- Gateway, Identity, and CareConnect are already working
- MySQL + EF Core are configured
- CareConnect already supports:
  - Providers
  - Referrals
  - Referral status history
  - Scheduling foundation
  - AppointmentSlots
  - Appointments
  - appointment lifecycle
  - ProviderAvailabilityExceptions
- Tenant context, role-based auth, audit fields, and exception handling already exist

Important:
This step should prepare the notes and attachment layer only.
Do NOT implement actual file upload/storage integration yet.
Do NOT integrate S3 or SynqDocs yet.
Do NOT add notifications or reminders yet.

PART 4 — VALIDATION
==================================================

14. Validation rules

Referral/Appointment notes:
- NoteType required and must be valid
- Content required, max 4000
- parent entity must exist in tenant

Attachment metadata:
- FileName required, max 500
- ContentType required, max 200
- FileSizeBytes >= 0
- Status must be valid
- parent entity must exist in tenant

Writes:

Keep health/info unchanged.
