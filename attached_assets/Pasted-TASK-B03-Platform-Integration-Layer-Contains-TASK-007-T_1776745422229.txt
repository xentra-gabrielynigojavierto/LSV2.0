TASK-B03 — Platform Integration Layer
(Contains TASK-007, TASK-008, TASK-009)

OBJECTIVE:
Evolve the standalone Task service into a platform-integrated service that is portable across LegalSynq products and orchestration-aware. This block must add cross-product context hardening, Flow linkage and event consumption, and external-consumer readiness without introducing Liens-specific assumptions.

NON-NEGOTIABLE RULES:
- Service name: task
- Database: tasks_db
- MUST remain standalone and reusable
- MUST preserve TASK-B01 and TASK-B02 behavior
- MUST NOT introduce Liens-specific runtime assumptions
- MUST support both:
  - product-scoped task visibility
  - general cross-product personal task aggregation
- MUST create report FIRST and update incrementally
- MUST validate against the current codebase before implementing
- MUST use existing platform patterns for auth, contracts, HTTP clients, event handling, logging, and configuration
- NO placeholders, fake integrations, or speculative architecture not grounded in the current codebase
