OBJECTIVE
==================================================
Enhance the existing LS-LIENS-FLOW-001 and LS-LIENS-FLOW-002 capabilities by introducing Event-Driven Task Generation for Synq Liens.
This feature must allow the system to automatically create tasks from selected business events using admin-managed task templates and tenant-configured automation rules.
Build:
1. Task Generation Rules
2. Template-based event-driven task creation
3. Duplicate prevention / idempotent generation behavior
4. Shared governance for automation rules across:
   - Tenant-side Synq Liens Product Settings
   - Control Center admin side
5. Generated task traceability
6. Case timeline visibility for generated tasks

==================================================
HIGH-LEVEL FEATURE SUMMARY
==================================================
Build a controlled event-to-task automation layer on top of FLOW-001 and FLOW-002.
A. Task Generation Rules
- tenant-defined rules that map a business event to a task template
- can be activated/deactivated
- can optionally override assignment behavior and due date behavior
B. Event-Driven Generation
- when supported Synq Liens events occur, the system evaluates active rules
- if a rule matches, a task is generated from the selected template
- generated task behaves like a normal task
C. Traceability
- generated tasks must store their automation source metadata
- users must be able to distinguish system-generated tasks from manual tasks
D. Shared Governance
- rules editable from:
  - Synq Liens → Product Settings → Task Automation
  - Control Center → Tenant → Synq Liens → Task Automation
- both surfaces manage the same underlying tenant rule set
