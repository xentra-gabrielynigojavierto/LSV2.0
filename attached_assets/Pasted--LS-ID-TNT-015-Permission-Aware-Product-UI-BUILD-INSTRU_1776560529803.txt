## FEATURE OBJECTIVE

Make product applications permission-aware so the UI reflects the backend authorization model by:
- hiding or disabling unavailable actions
- supporting read-only states where appropriate
- reducing avoidable 403 dead ends
- standardizing permission-denied UX

This feature should align product UX with the backend permission enforcement delivered in LS-ID-TNT-012.
