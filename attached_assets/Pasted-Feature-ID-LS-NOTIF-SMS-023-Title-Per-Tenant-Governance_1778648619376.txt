Feature ID: LS-NOTIF-SMS-023

Title:
Per-Tenant Governance Rule Pack Scoping and True Tenant-Isolated Enforcement

Objective:
Introduce true tenant-scoped governance enforcement by enabling tenant-isolated rule-pack activation, tenant-specific governance overlays, scoped governance resolution, segmented rollout enforcement, tenant-aware governance execution, and tenant-specific rollback semantics within the SMS governance engine.

Critical Architectural Constraint:
Notification Service remains the source of truth for:
- tenant-scoped governance rule activation
- tenant-to-rule-pack assignment
- tenant governance overlays
- tenant-aware governance resolution
- effective governance graph calculation
- rollout-cohort enforcement isolation
- tenant-specific governance rollback
- tenant governance APIs

Control Center must only consume Notification Service APIs and render/administer tenant governance visibility. Do not duplicate tenant governance resolution, overlay merging, assignment activation, or enforcement logic in Control Center.

Requirements:
- Implement tenant-scoped governance enforcement inside Notification Service.
- Preserve LS-NOTIF-SMS-017 governance policies.
- Preserve LS-NOTIF-SMS-018 template governance.
- Preserve LS-NOTIF-SMS-019 dynamic rules.
- Preserve LS-NOTIF-SMS-020 versioning, rollback, import/export, and analytics.
- Preserve LS-NOTIF-SMS-021 release management.
- Preserve LS-NOTIF-SMS-021-HARDENING activation locking, approval, audit, retry/backoff, and integrity behavior.
- Preserve LS-NOTIF-SMS-022 rollout orchestration.
- Tenant-aware governance must extend, not replace, existing global governance behavior.
- Existing global-only tenants must behave exactly as before.
- Tenant-specific activation/rollback must not affect unrelated tenants.
- Governance resolution must remain fast, safe, and fail-open where configured.
- Do not store raw phone numbers in tenant governance assignment/overlay tables.
- Do not expose provider credentials, webhook URLs, SettingsJson, CredentialsJson, auth tokens, raw provider payloads, or recipient phone numbers.
- Keep Control Center as admin/visibility UX only.
- Preserve all LS-NOTIF-SMS-001 through LS-NOTIF-SMS-022 behavior.

Implementation Requirements:
1. Add tenant rule-pack assignment persistence.
2. Add tenant governance overlay persistence.
3. Add tenant-aware governance resolution service.
4. Add tenant assignment orchestration service.
5. Add tenant isolation validation service.
6. Extend dynamic rule resolver from LS-NOTIF-SMS-019 to resolve effective rules by tenant.
7. Extend rollout behavior from LS-NOTIF-SMS-022 so canary/staged cohorts can activate governance for targeted tenants only.
8. Add tenant-specific rollback/deactivation semantics.
9. Add effective governance graph APIs.
10. Add tenant assignment/overlay admin APIs.
11. Add Control Center tenant governance assignment UI if feasible.
12. Ensure tenant-aware resolution remains backward compatible with existing global behavior.
13. Ensure all failures degrade safely.
