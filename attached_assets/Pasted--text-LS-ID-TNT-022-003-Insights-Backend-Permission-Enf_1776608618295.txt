# LS-ID-TNT-022-003 — Insights Backend Permission Enforcement

## FEATURE OBJECTIVE

Add authoritative backend permission enforcement for the Insights product so the UI gating delivered in LS-ID-TNT-022-002 is matched by real API-level security.

This feature wires the new `SYNQ_INSIGHTS.*` permission codes into the Reports / Schedules backend so direct API calls, stale-session actions, and non-UI access paths are correctly denied server-side.

This is the final step that turns Insights authorization from:
- permission-modeled
- UI-gated

into:

- permission-modeled
- UI-gated
- backend-enforced

---

## WHY THIS FEATURE COMES NEXT

Current state:
- LS-ID-TNT-022-001 defined the Insights permission catalog
- LS-ID-TNT-022-002 wired all major Insights client components to those permissions
- But the backend still does **not** enforce `Insights.*` permission policies

That means:
- the UI behaves correctly
- but direct API calls can still bypass those UI controls

This feature closes that security gap.
