# LS-ID-TNT-016-01 — Tenant-Subdomain User Management Email URL Alignment

## FEATURE OBJECTIVE

Adjust all outgoing **user-management-related emails** so links use the tenant-aware format:

[tenant-subdomain].[baseurl]

This applies especially to:
- user invitation emails
- invite resend emails
- password reset emails
- any other identity/user-management emails that currently rely on a generic portal base URL

The goal is to ensure that all user-management email links route users into the correct tenant-branded portal context instead of a shared or localhost-style base URL.

---

## WHY THIS FEATURE COMES NEXT

Current state from the verified invite/activation implementation:
- invitation email flow already exists
- password reset email flow already exists
- notifications service integration already exists
- activation page already exists

The remaining issue is URL correctness and tenancy alignment.

Right now, user-management emails rely on a general portal base URL setting. That is not sufficient if the required routing model is:

[tenant-subdomain].[baseurl]

This feature makes email links tenant-correct and production-usable.

## REQUIRED URL FORMAT

Use this exact pattern for user-management links:

https://[tenant-subdomain].[baseurl]/...

Examples:
- invite acceptance:
  https://acme.portal.legalsynq.net/accept-invite?token=...
- password reset:
  https://acme.portal.legalsynq.net/reset-password?token=...
- other user-management activation/recovery pages:
  https://acme.portal.legalsynq.net/<path>

The implementation must derive:
- tenant-subdomain from tenant data
- baseurl from configuration

Do NOT hardcode localhost or a shared non-tenant portal URL for production paths.
