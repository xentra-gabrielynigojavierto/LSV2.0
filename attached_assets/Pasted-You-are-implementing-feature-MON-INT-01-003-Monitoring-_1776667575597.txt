You are implementing feature MON-INT-01-003 — Monitoring Auth & Security Alignment.

OBJECTIVE
Align the Monitoring Service authentication and security model with the main LegalSynq platform so that platform components can use the Monitoring admin/write path without relying on temporary bootstrap workarounds, while preserving safe access to monitoring read endpoints.

This feature replaces the current auth mismatch (Monitoring RS256 vs platform HS256) with a coherent platform-aligned security model.

ARCHITECTURE RULE (NON-NEGOTIABLE)
- Monitoring Service = source of truth
- Control Center = read-only consumer
- Monitoring write/admin operations must be secured
- Monitoring read operations may remain platform-consumable, but the access model must be explicit and intentional
- Do NOT weaken security by leaving sensitive write surfaces anonymously accessible

MANDATORY EXECUTION RULES
- Create the report file FIRST at: /analysis/MON-INT-01-003-report.md
- Update the report incrementally after each meaningful step
- Do NOT wait until the end to write the report
- Do NOT introduce placeholder auth
- Do NOT leave the system in a half-switched auth state
- Do NOT break existing read endpoints without documenting and validating the replacement access path
- Prefer incremental, reversible changes
- Keep backend, gateway, and consumer concerns clearly separated

CURRENT CONTEXT
- Monitoring Service is integrated and operational
- Read model is implemented
- Entity bootstrap is currently done through an internal hosted-service workaround
- Current gap: Monitoring Service uses RS256 with a different issuer/audience than the rest of the platform
- Platform services use the standard platform auth pattern and cannot call Monitoring admin endpoints today
- This feature should remove that mismatch cleanly

GOAL OF THIS FEATURE
By the end of this feature:
- Monitoring admin endpoints must be callable through a platform-aligned auth model
- The temporary bootstrap workaround should be retired or clearly marked for retirement
- Monitoring read and write surfaces must have an intentional security posture
- The chosen approach must be validated end-to-end
