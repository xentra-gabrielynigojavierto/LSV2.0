## 1. Version Fallback Strategy (CRITICAL FIX)

### Problem

Current fallback:

```text
Redis failure → local Interlocked counter
```

This causes:

* cross-node divergence
* inconsistent authorization decisions

---

### Required Fix

Replace fallback behavior:

#### Option A (RECOMMENDED — Freeze Mode)

* On Redis failure:

  * **DO NOT increment locally**
  * continue using last known version
  * disable cache writes
  * allow reads (best effort)

#### Option B (Strict Mode)

* disable caching entirely when version provider unavailable

---

### Required Behavior

* version must never diverge across nodes
* degraded mode must be explicitly logged

---

### Logging

```text
PolicyVersionFallbackMode: Freeze | DisabledCache
```

---

## 2. Cache Stampede Protection

### Problem

On version change:

* all nodes miss cache
* multiple identical evaluations executed

---

### Required Implementation

Introduce **per-key request coalescing**

### Pattern

```text
cache key → lock → single evaluation → shared result
```

---

### Implementation Options

* `ConcurrentDictionary<string, SemaphoreSlim>`
* async locking per cache key

---

### Requirements

* only one evaluation per key at a time
* other requests await result
* no deadlocks
* lock cleanup after use

---

## 3. Tenant-Scoped Versioning (Optional but Designed)

### Problem

Global version invalidates all tenants.

---

### Enhancement

Support:

```text
policy:{tenantId}:version
```

---

### Required

* abstraction must support both:

  * global version
  * tenant-scoped version

---

### Config

```json
{
  "PolicyVersioning": {
    "Scope": "Global | Tenant"
  }
}
```

---

### Default

* Global (preserve current behavior)
