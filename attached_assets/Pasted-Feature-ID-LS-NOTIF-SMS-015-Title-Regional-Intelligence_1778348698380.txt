STEP 5 — Add Quality Calculation Service
Add service to calculate provider quality snapshots from local telemetry.

Suggested interface:
ISmsProviderQualityService

Suggested methods:
- CalculateSnapshotsAsync(DateTime windowStart, DateTime windowEnd, CancellationToken ct)
- GetLatestSnapshotAsync(providerType, tenantId, providerConfigId, countryCode, ct)
- QuerySnapshotsAsync(query, ct)
- GetProviderScoresAsync(query, ct)

Calculation inputs:
- NotificationAttempt.Channel == sms
- Provider
- ProviderConfigId
- ProviderOwnershipMode
- TenantId
- Status
- AttemptNumber / IsFailover
- CreatedAt / CompletedAt
- LastReconciliationOutcome
- ReconciliationAttemptCount
- EstimatedCostAmount / ActualCostAmount

Suggested calculations:
- DeliverySuccessRate = delivered/successful-or-total depending existing status semantics; document exact rule.
- FailureRate = failed + dead_letter / total.
- RetryRate = attempts with AttemptNumber > 1 or IsFailover / total.
- ReconciliationFailureRate = reconciliation failures / reconciled attempts.
- AverageLatencyMs = CompletedAt - CreatedAt where both available.
- AverageEffectiveCost = average ActualCost ?? EstimatedCost.
- CostPerDeliveredMessage = total effective cost / delivered count.
- QualityScore = weighted score bounded 0–100.
- CostEfficiencyScore = normalized score where lower cost per delivered is better.

Requirements:
- Use local data only.
- No external provider calls.
- No SMS sends/retries/reconciliation.
- Must handle zero attempts safely.
- Must respect MinimumAttemptCount.
- Persist snapshots.
- Avoid unbounded queries.
- Use UTC consistently.

STEP 7 — Add Adaptive Routing Modes
Extend routing mode support.

New modes:
- adaptive_quality
- adaptive_balanced
- adaptive_regional

Mode behavior:

adaptive_quality:
- Select provider with highest latest QualityScore among candidates.
- If no candidate has sufficient data, fall back to priority mode.

adaptive_balanced:
- Combine:
  - quality score
  - effective estimated cost
  - health penalty
  - failure/retry/reconciliation penalty
- Select best composite score.
- If data is insufficient, fall back to hybrid or priority.

adaptive_regional:
- Infer recipient country/region.
- Prefer provider with best quality score for that country/region.
- If no regional score exists, fall back to adaptive_quality.
- If no quality score exists, fall back to priority.

Requirements:
- Existing modes remain unchanged.
- Candidate provider set must still come from ProviderRoutingService / tenant config routes.
- Adaptive routing must not select excluded providers.
- Adaptive routing must respect MaxEstimatedCostPerMessage if configured.
- Adaptive routing must respect RequireHealthyProvider if configured.
- Routing engine must persist decision reason explaining fallback or selected score.
- No external provider calls during route selection.
