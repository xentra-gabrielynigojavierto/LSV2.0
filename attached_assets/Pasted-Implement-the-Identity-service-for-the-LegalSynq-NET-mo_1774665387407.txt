Implement the Identity service for the LegalSynq .NET monorepo using .NET 8, EF Core, and MySQL.

Context:
The monorepo already contains these projects:
- apps/services/identity/Identity.Api
- apps/services/identity/Identity.Application
- apps/services/identity/Identity.Domain
- apps/services/identity/Identity.Infrastructure
- shared/contracts/Contracts
- shared/building-blocks/BuildingBlocks

Goal:
Build the initial Identity data foundation with this exact schema shape:
- Tenants
- Users
- Roles
- UserRoles
- Products
- TenantProducts

Use clean architecture boundaries:
- Domain: entities only
- Infrastructure: EF Core, entity configurations, DbContext, seed data
- Api: DI registration, endpoints, startup
- Application: keep minimal for now, but keep references correct

Requirements

1) Add NuGet packages
Add the latest stable versions compatible with .NET 8 for:
- Microsoft.EntityFrameworkCore
- Microsoft.EntityFrameworkCore.Design
- Pomelo.EntityFrameworkCore.MySql

Add them only where needed:
- Identity.Infrastructure
- Identity.Api if needed for migrations startup

2) Create domain entities in Identity.Domain

Create these classes:
- Tenant
- User
- Role
- UserRole
- Product
- TenantProduct

Use Guid keys consistently where specified.
Use UTC timestamps.

Entity definitions:

- Id : Guid
- Name : string

- CreatedAtUtc : DateTime
- UpdatedAtUtc : DateTime

User
- Id : Guid
- TenantId : Guid

- LastName : string
- IsActive : bool
- CreatedAtUtc : DateTime
- UpdatedAtUtc : DateTime

Role
- Id : Guid

- Name : string
- Description : string?

- CreatedAtUtc : DateTime
- UpdatedAtUtc : DateTime

UserRole
- UserId : Guid

- AssignedAtUtc : DateTime

Product
- Id : Guid

- Description : string?
- IsActive : bool

TenantProduct
- TenantId : Guid
- ProductId : Guid
- IsEnabled : bool

3) Create IdentityDbContext in Identity.Infrastructure

Create IdentityDbContext and map all entities using Fluent API only.
Use separate IEntityTypeConfiguration<T> classes for each entity.

Table names must be exactly:
- Tenants
- Users
- Roles
- UserRoles
- Products
- TenantProducts

Relationships:
- User -> Tenant (many-to-one)
- Role -> Tenant (many-to-one)
- UserRole -> User (many-to-one)
- UserRole -> Role (many-to-one)
- TenantProduct -> Tenant (many-to-one)
- TenantProduct -> Product (many-to-one)

Primary keys:
- Tenants: Id
- Users: Id
- Roles: Id
- Products: Id
- UserRoles: composite key (UserId, RoleId)
- TenantProducts: composite key (TenantId, ProductId)

Indexes:
- Tenant.Code unique
- User unique on (TenantId, Email)
- Role unique on (TenantId, Name)
- Product.Code unique

Use sensible lengths for strings, for example:
- Name 200
- Code 100
- Email 320
- PasswordHash 500
- FirstName 100
- LastName 100
- Description 1000

Update appsettings.json and appsettings.Development.json in Identity.Api.

Use a placeholder connection string like:

Do not hardcode secrets anywhere else.
