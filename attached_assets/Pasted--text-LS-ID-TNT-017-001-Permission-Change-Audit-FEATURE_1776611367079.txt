# LS-ID-TNT-017-001 — Permission Change Audit

## FEATURE OBJECTIVE

Implement a foundational audit layer to track all **permission-related changes** across the platform, including:
- role assignments
- role removals
- permission mapping changes
- product access changes
- group membership changes (if they affect permissions)

This feature establishes the **authoritative audit trail** required for:
- compliance
- governance
- operational traceability
- future audit UI (LS-ID-TNT-017-004)

This is a **backend-first audit capture feature**. It does NOT include full audit UI or reporting yet.

---

## WHY THIS FEATURE COMES NEXT

Current system:
- authorization model is complete
- UI + backend enforcement is complete
- real-time sync is complete

Missing:
- **traceability**

Right now:
- changes happen
- but there is no persistent record of:
  - who made the change
  - what changed
  - when it changed
  - what the previous state was

This is required for:
- enterprise compliance
- debugging access issues
- operational trust

---

## HARD SCOPE BOUNDARY (DO NOT VIOLATE)

DO NOT IMPLEMENT:
- full audit UI (that is LS-ID-TNT-017-004)
- analytics/reporting dashboards
- event streaming infrastructure
- external logging platforms (Datadog, etc.)
- audit for non-permission domains (cases, financials, etc.)
- ABAC/policy engine
- speculative fields not grounded in current data

ONLY IMPLEMENT:
- audit data model (DB)
- audit capture hooks in permission-changing services
- consistent audit event schema
- minimal backend utilities for logging audit entries
- report-first implementation

## STEP 2 — DESIGN AUDIT DATA MODEL

Create a dedicated audit table.

### Table: `idt_AuditLogs` (or equivalent)

Required fields:

| Field | Type | Description |
|------|------|-------------|
| Id | GUID / BIGINT | Primary key |
| TenantId | GUID | Tenant scope |
| ActorUserId | GUID | Who made the change |
| TargetUserId | GUID (nullable) | User affected |
| ActionType | string | e.g. ROLE_ASSIGNED |
| EntityType | string | USER / GROUP / TENANT |
| EntityId | GUID | ID of affected entity |
| Changes | JSON | Before/After snapshot |
| TimestampUtc | datetime | Event time |
| CorrelationId | string | Optional request trace |

Requirements:
- JSON must capture **before and after state**
- table must be indexed on TenantId + Timestamp
- ensure no cross-tenant leakage
