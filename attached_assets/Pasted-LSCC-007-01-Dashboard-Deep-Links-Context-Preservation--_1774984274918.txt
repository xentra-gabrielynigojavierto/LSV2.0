==================================================
REQUIREMENTS
==================================================

--------------------------------------------------
A. DASHBOARD DEEP LINKS
--------------------------------------------------

Update dashboard links to include context parameters.

Minimum links to update:
- referral activity cards
- “Sent Referrals” navigation
- “Referral Inbox” navigation

Required query params:

- from=dashboard

Add additional params where meaningful:
- status (e.g., New, Accepted)
- existing date range params if already present

Examples:

- /careconnect/referrals?from=dashboard
- /careconnect/referrals?from=dashboard&status=New
- /careconnect/referrals?from=dashboard&status=Accepted

Rules:
- do NOT invent new param names unnecessarily
- reuse existing param names used by referral list

--------------------------------------------------
B. REFERRAL LIST → DETAIL CONTEXT PRESERVATION
--------------------------------------------------

When linking from referral list to detail page:

Preserve relevant query params:

- from
- status
- search
- any existing date-related params

Implementation:

- append current query string to detail link
- or use a helper utility to build links

Example:

/careconnect/referrals/{id}?from=dashboard&status=New&search=John

Rules:
- must be deterministic
- must not drop existing params
- must not introduce malformed URLs

--------------------------------------------------
C. REFERRAL DETAIL BACK NAVIGATION
--------------------------------------------------

Improve back-link behavior on:

/careconnect/referrals/{id}

Back navigation priority:

1. If list context exists (status/search/etc):
   → Back to filtered referrals list

2. If only from=dashboard:
   → Back to dashboard

3. Fallback:
   → Back to generic referrals list

Implementation guidance:

- inspect query params
- build target URL accordingly
- do NOT rely on browser history

Examples:

Case 1:
?from=dashboard&status=New
→ /careconnect/referrals?status=New&from=dashboard

Case 2:
?from=dashboard
→ /careconnect/dashboard

Case 3:
(no params)
→ /careconnect/referrals

--------------------------------------------------
D. CONTEXT-AWARE COPY (LIGHTWEIGHT)

Update back link labels where useful:

Examples:

- "← Back to Dashboard"
- "← Back to Sent Referrals"
- "← Back to Referral Inbox"
- "← Back to Referrals"

Rules:
- keep simple
- no complex conditional UI
- optional but preferred for clarity

E. HELPER UTILITIES (OPTIONAL)
