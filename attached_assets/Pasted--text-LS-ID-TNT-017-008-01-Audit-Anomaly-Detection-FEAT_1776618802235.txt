# LS-ID-TNT-017-008-01 — Audit Anomaly Detection

## FEATURE OBJECTIVE

Implement a practical **Audit Anomaly Detection** layer that derives anomaly signals from the existing audit analytics and event store, so the platform can identify unusual audit activity patterns without requiring machine learning or external tooling.

This feature should detect and surface anomalies such as:
- denial spikes
- unusual actor activity spikes
- unusual tenant activity spikes
- abnormal governance activity bursts
- unusually high audit export/query activity
- unusual event-type concentration over a recent time window

This is a **rule-based anomaly detection feature**, not predictive analytics and not a full alerting/notification engine.

Current state:
- audit capture is complete
- audit viewer is complete
- audit correlation is complete
- audit analytics is complete

What is missing:
- operators can see counts and trends
- but the system does not yet identify when something is statistically or operationally unusual

Right now, anomaly recognition is still manual.

This feature converts the platform from:
- **audit insight**
into:
- **proactive audit intelligence**
