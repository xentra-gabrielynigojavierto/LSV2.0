You are implementing LS-LIENS-UI-003 for the LegalSynq Liens v2 migration.

Feature ID: LS-LIENS-UI-003  
Feature Title: Cross-Entity Cases ↔ Liens Integration with Enforced Service Layer Pattern

Objective:
Wire the Liens module to real backend APIs and establish full cross-entity integration between Cases and Liens. This must strictly follow the frontend architecture pattern established in LS-LIENS-UI-002 and enable real navigation and data consistency between Cases and Liens.

This is the first cross-entity integration feature and must serve as the standard for all future domain relationships.

---

# 🔴 Critical Architecture Rules (Non-Negotiable)

1. Preserve the v2 shell (top nav + side nav).
2. Preserve existing UI layout and flow from UI-01 unless minimal adaptation is required.
3. Enforce strict layering:
   - API client layer
   - Service layer
   - Mapper/adapter layer
   - UI state layer
   - Presentation layer
4. NO direct HTTP calls in pages/components.
5. NO DTO-to-UI transformation inside components.
6. NO production mock/static lien data.
7. MUST reuse the exact architectural pattern from LS-LIENS-UI-002.
8. MUST NOT invent backend APIs — use only what exists.
9. MUST preserve donor SynqLiens behavior where supported.
10. Cross-entity data must come ONLY from backend truth.

---

# 🧱 Required Frontend Architecture

You MUST replicate the same structure used in Cases:

Layer 1 — API Client (`liens.api.ts`)
- raw HTTP only
- endpoints, params, headers
- typed DTOs
- NO UI shaping

Layer 2 — Service Layer (`liens.service.ts`)
- business-facing methods
- only entry point for UI
- calls API client
- applies mappers

Layer 3 — Mapper Layer (`liens.mapper.ts`)
- DTO → UI model transformation
- normalization
- status/display mapping
- NO duplication across components

Layer 4 — Types (`liens.types.ts`)
- DTOs
- query models
- UI models
- cross-entity types (minimal)

Layer 5 — UI State (Zustand or existing pattern)
- UI state only (filters, pagination, selection, loading)
- NOT source of truth for data

Layer 6 — Presentation (pages/components)
- rendering only
- receives typed models
- no HTTP
- no raw response handling
