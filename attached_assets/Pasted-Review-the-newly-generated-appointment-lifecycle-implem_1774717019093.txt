==================================================

- Cancelled terminal
- NoShow terminal

Check that no extra transitions were introduced.

4. CANCELLATION LOGIC REVIEW
==================================================

Verify cancellation behavior:
- tenant-scoped
- only valid from non-terminal states
- writes AppointmentStatusHistory
- decrements old slot ReservedCount safely
- does not allow ReservedCount below 0
- may reopen slot if capacity becomes available again
- booking changes and history are saved together

==================================================
5. RESCHEDULE LOGIC REVIEW
==================================================

Verify reschedule behavior:
- tenant-scoped
- only allowed from Scheduled or Confirmed
- new slot must exist in tenant
- new slot must be Open
- new slot must have available capacity
- old slot ReservedCount decremented safely
- new slot ReservedCount incremented safely
- appointment times/provider/facility/service copied from new slot
- no status history row created if status does not change
- save is atomic
