You are implementing feature MON-INT-03-001 — Incident / Alert Detail View.

OBJECTIVE
Add a read-only Incident / Alert Detail View to the Control Center monitoring page so operators can open an alert and immediately see the key incident context in one place: severity, message, affected component, timestamps, and current related component status.

This is the next usability layer after the Status Summary Banner and Component Status List.

ARCHITECTURE RULE (NON-NEGOTIABLE)
- Monitoring Service = source of truth
- Control Center = read-only consumer
- The detail view must use existing monitoring read data
- Do NOT introduce mutation flows
- Do NOT modify Monitoring Service unless an absolute blocker is proven
- Do NOT reintroduce monitoring logic into Control Center

MANDATORY EXECUTION RULES
- Create the report file FIRST at: /analysis/MON-INT-03-001-report.md
- Update the report incrementally after each meaningful step
- Do NOT wait until the end to write the report
- Do NOT add acknowledge / resolve actions
- Do NOT add fake incident fields
- Prefer minimal, reversible UI changes
- Keep existing banner, component list, and alerts panel intact
- Support both MONITORING_SOURCE=local and MONITORING_SOURCE=service
