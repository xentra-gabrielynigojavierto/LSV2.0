--------------------------------------------------

C.1 Provider Activation UX

When user clicks referral link:

Flow:

1. User lands on activation route:
   `/activate?token=...`

2. Frontend calls:
   `POST /api/careconnect/referrals/{id}/auto-provision`

3. Backend:
   - validates token
   - provisions org + user
   - sends invitation

4. Frontend behavior:
   - show success state
   - redirect to login

--------------------------------------------------

C.2 Post-Login Redirect

After login:

- redirect to:
  `/careconnect/referrals/{id}`

--------------------------------------------------

C.3 Existing User Path

If user already exists:

- skip provisioning
- redirect directly to login
- preserve `returnTo`
