# LS-TENANT-001 — Tenant Admin Foundation + Navigation

## Objective

Establish the **foundational tenant admin experience layer** in the tenant portal, including:

* tenant admin role enforcement
* authorization-aware navigation
* route protection
* base layout for authorization modules
* structural groundwork for all LS-TENANT features

This feature creates the **entry point and skeleton** for tenant authorization management.

## Architectural Principles

### Tenant Isolation

* all UI and routes must operate strictly within the current tenant context
* no cross-tenant access paths

### Backend-Driven Authorization

* do not rely on frontend-only role checks
* backend (Identity / LS-COR-AUT) remains source of truth

### Progressive Enhancement

* this feature creates structure only
* detailed functionality comes in LS-TENANT-002+

---

## In Scope

---

## 1. Tenant Admin Role Enforcement

### Required Behavior

Only users with **TenantAdmin (or equivalent)** role may access tenant authorization UI.

---

### Implementation

#### Frontend Guard

* implement route-level guard:

  * check user roles from JWT / session
  * must include:

    ```text
    TenantAdmin
    ```
  * optionally support:

    ```text
    PlatformAdmin (if allowed in tenant portal)
    ```

* if unauthorized:

  * redirect to:

    * dashboard OR
    * access denied page

---

#### Backend Validation

* all protected routes must call backend APIs that:

  * enforce tenant boundary
  * enforce role/permission

---

### Acceptance

* non-admin users cannot access `/tenant/authorization/*`
* direct URL access is blocked

---
