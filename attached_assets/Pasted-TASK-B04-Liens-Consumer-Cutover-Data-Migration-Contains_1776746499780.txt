TASK-B04 — Liens Consumer Cutover & Data Migration
(Contains TASK-010, TASK-011, TASK-012)

OBJECTIVE:
Transition Synq Liens from owning task runtime to consuming the standalone Task service. This includes safe data migration, dual-write strategy, read cutover, and removal of Liens-owned task logic—without breaking the tenant portal experience.

NON-NEGOTIABLE RULES:
- Task service remains the single source of truth for all task runtime
- Liens must become a consumer ONLY (no task persistence or business logic ownership)
- ZERO breaking changes to tenant portal UX
- MUST follow incremental migration (no big-bang switch)
- MUST create report FIRST and update incrementally
- MUST validate against current codebase before changes
- MUST support rollback at every stage
- NO placeholders or simulated migration logic
