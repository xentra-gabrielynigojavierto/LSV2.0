PROJECT: NOTIF-UI-007 — Tenant Template Visibility (Read-Only)
==================================================

GOAL:
Provide tenant users with read-only visibility into notification templates that apply to them, scoped strictly by product type, including branding-aware preview.

This phase is READ-ONLY.

Do NOT allow editing, overriding, or publishing.

==================================================
CRITICAL DOMAIN RULE — PRODUCT FIRST (MANDATORY)
==================================================

Templates are strictly product-scoped.

The UI MUST enforce a product-first access pattern.

This is NOT optional.

Requirements:

1. A product MUST be selected BEFORE any templates are displayed
2. Templates must NEVER be shown across multiple products at once
3. All template queries MUST include productType
4. The UI must clearly indicate the active product context at all times

Allowed UX patterns:

Preferred (RECOMMENDED):
- Route-based:
  /tenant/notifications/templates/[productType]

Example:
  /tenant/notifications/templates/careconnect

Alternative:
- Product selector (tabs/dropdown) with enforced selection state

DO NOT:
- load templates without productType
- show cross-product template lists
- allow ambiguous product context

==================================================
2. TEMPLATE LIST (PRODUCT-SCOPED)
==================================================

Route:
- /tenant/notifications/templates/[productType]

Backend:
- GET /notifications/v1/templates/global?productType={productType}

Display ONLY templates for that product.

Columns / cards:
- templateKey
- name
- channel
- current version indicator
- last updated
- optional category

UX:
- clean table or card list
- empty state if none exist
- show active product clearly at top

==================================================
3. TEMPLATE DETAIL VIEW
==================================================

Route:
- /tenant/notifications/templates/[productType]/[templateId]

Backend:
- GET /notifications/v1/templates/global/:id
- GET /notifications/v1/templates/global/:id/versions

Display:
- template metadata
- product type (read-only)
- channel
- versions list

Versions list:
- version number
- status (draft / published)
- created at
- published at
- highlight current version

Actions:
- View Version
- Preview Version

NO EDIT BUTTONS

==================================================
4. TEMPLATE VERSION VIEW
==================================================

Display selected version content:

- subject (if email)
- HTML content (render safely)
- text content
- variables schema (if available)
- sample data (if available)

Also display:
- note that content is controlled by platform
- tenant cannot modify

==================================================
5. BRANDED PREVIEW (CRITICAL FEATURE)
==================================================

Backend:
- POST /notifications/v1/templates/global/:id/versions/:versionId/preview

Payload:
- tenantId (from context)
- productType (from route)
- templateData (user input)

UI Requirements:

Preview Panel must show:
- subject
- rendered HTML
- text version

Preview Input:
- dynamic form based on variables
- allow user to input sample values

Branding Awareness:
- show which branding is applied:
  - brand name
  - logo (if available)
  - colors summary

Fallback Behavior:
- if tenant branding does not exist:
  show:
    "Default branding applied"

IMPORTANT:
- preview must use REAL backend call
- do NOT simulate preview
