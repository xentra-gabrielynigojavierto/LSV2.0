TASK-MIG-05 — Template + Stage Compatibility Alignment

OBJECTIVE:
Align the already-migrated Liens template and stage configuration now hosted in the Task service so template applicability, start-stage behavior, contextual filtering, and generation compatibility remain correct as the system progressively shifts from Liens-owned config reads to Task-owned config reads.

This step is an **alignment step**, not a new feature build.

NON-NEGOTIABLE RULES:
- DO NOT redesign generation rules
- DO NOT migrate Flow orchestration logic
- DO NOT collapse Task stages into Flow stages
- DO NOT delete any Liens tables in this step
- DO NOT remove fallback paths in this step
- ZERO UI behavior regression
- MUST preserve current Liens generation and task-board behavior
- MUST support rollback
- MUST create report FIRST and update incrementally
- MUST inspect actual code/schema before implementation
- NO placeholders

CRITICAL BOUNDARY:
- Template applicability to stages in this prompt refers to **Liens task-board stages / Task stage configs**
- It does NOT refer to Flow orchestration stages
- Any `ApplicableWorkflowStageId` currently carried by Liens templates remains a task-board stage concept during this migration phase
