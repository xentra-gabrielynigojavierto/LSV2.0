4. FLOW API INTEGRATION TEST MATRIX

You MUST add automated Flow API integration tests for:

A. Ownership / IDOR
- GET by wrong parent context fails closed
- ADVANCE by wrong parent context fails closed
- COMPLETE by wrong parent context fails closed

B. Tenant isolation
- tenant A cannot read or mutate tenant B workflow instance
- service token with tenant A cannot mutate tenant B workflow instance

C. Product correlation
- SynqLien context cannot operate CareConnect-owned mapping
- CareConnect context cannot operate SynqFund-owned mapping
- SynqFund context cannot operate SynqLien-owned mapping

D. Transition integrity
- stale expectedCurrentStepKey returns 409 with expected_step_mismatch
- inactive instance advance/complete returns 409 with instance_not_active
- concurrent mutation path returns 409 with concurrent_state_change if feasible in test harness

E. Auth
- user with valid capability succeeds where expected
- user without capability is denied
- invalid issuer/audience/service-token claims are denied
- missing tenant claim is denied

F. Error contract
- responses return the expected structured error code
- no sensitive ownership detail leaks in denial responses
