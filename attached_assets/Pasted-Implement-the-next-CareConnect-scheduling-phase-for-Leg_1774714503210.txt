Implement the next CareConnect scheduling phase for LegalSynq: AppointmentSlots + slot generation + Appointments.

Goal:
Turn the existing scheduling foundation into a bookable scheduling model by adding:
1. AppointmentSlots
2. slot generation from ProviderAvailabilityTemplates
3. Appointments

Context:
- Gateway, Identity, and CareConnect are already working
- MySQL + EF Core are already configured
- CareConnect already supports:
  - Providers
  - Categories
  - Referrals
  - Referral status history
  - workflow rules
  - Facilities
  - ServiceOfferings
  - ProviderAvailabilityTemplates
- Tenant context, role-based auth, audit fields, and exception handling already exist

Important:
This step should create the initial bookable scheduling model only.
Do NOT implement yet:
- scheduling exceptions / blackout dates
- rescheduling
- recurring booking workflows
- appointment status history
- notifications
- documents
- payments
- geo search

PART 3 — SLOT GENERATION
==================================================

6. Implement slot generation service

Create a service such as:
- ISlotGenerationService
- SlotGenerationService

Purpose:
Generate bookable AppointmentSlots from ProviderAvailabilityTemplates for a requested date range.

Input:
- ProviderId
- FromDateUtc
- ToDateUtc

Behavior:
- load active ProviderAvailabilityTemplates for the provider and tenant
- for each day in range, match templates by DayOfWeek
- create slots between StartTimeLocal and EndTimeLocal
- use SlotDurationMinutes
- set Capacity from template
- set ReservedCount = 0
- set Status = Open
- set ProviderAvailabilityTemplateId
- set ServiceOfferingId if template has one

Important:
- generate slots in UTC using the current simplified assumption
- do not solve full time zone complexity yet
- avoid duplicate slot creation if generation is run twice for the same provider/date range/template
- keep generation tenant-scoped

7. Add slot generation endpoint

Implement:
- POST /api/providers/{id}/slots/generate

Request DTO:
- FromDateUtc : DateTime
- ToDateUtc : DateTime

Authorization:
- PlatformOrTenantAdmin only

Validation:
- ToDateUtc >= FromDateUtc
- range should be reasonable (for example max 60 days in one request)
- provider must exist in tenant

Response:
- count of slots created
- provider id
- date range

PART 5 — APPOINTMENT BOOKING
==================================================

9. Add appointment creation endpoint

Implement:
- POST /api/appointments

Request DTO:
- ReferralId
- AppointmentSlotId
- Notes

Behavior:
- tenant-scoped
- referral must exist in tenant
- slot must exist in tenant
- slot must be Open
- ReservedCount must be < Capacity
- create Appointment
- set ScheduledStartAtUtc and ScheduledEndAtUtc from slot
- set ProviderId / FacilityId / ServiceOfferingId from slot
- set Status = Scheduled
- increment slot ReservedCount by 1
- if slot becomes full, either keep Status = Open with ReservedCount=Capacity or set to Closed; choose one simple rule and keep it consistent
- save appointment and slot update together

PART 6 — VALIDATION AND BUSINESS RULES
==================================================

11. Validation rules

Slot generation:
- provider must exist
- date range valid
- date range not excessive

Slot listing:
- page >= 1
- pageSize >= 1
- pageSize <= 100

Appointment creation:
- ReferralId required
- AppointmentSlotId required
- slot must be Open
- slot must have remaining capacity
- referral and slot must belong to same tenant

12. Business rules for this phase
- no double creation of identical slots for the same provider/template/time
- appointments are initially created only from a slot
- one slot can hold multiple appointments only up to Capacity
- do not implement rescheduling or appointment status transitions yet
