# LS-COR-AUT-011 — Advanced Authorization (ABAC + Context-Aware Policies)

## Objective

Extend LS-COR-AUT from **permission-based authorization (PBAC)** to **policy-driven authorization (ABAC + contextual enforcement)** by introducing:

* attribute-based access control (ABAC)
* contextual policy evaluation
* conditional permissions
* dynamic enforcement rules

At completion, the system must support:

```text
User → Roles → Permissions → Policies → Context → Enforcement
```

This enables **fine-grained, dynamic, real-world authorization decisions** beyond static roles and permissions.

---

## Architectural Principles

### Layered Authorization Model

* RBAC → assignment
* PBAC → capability
* ABAC → decision logic

All layers must coexist.

---

### Non-Breaking Evolution

* Existing permission enforcement remains functional
* Policies are additive (not replacing permissions initially)

---

### Explicit Evaluation

* Authorization decisions must be:

  * explainable
  * traceable
  * observable

---

## In Scope

---

## 1. Policy Model (Core)

### Create Entity

`Policy`

Fields:

* `Id`
* `PolicyCode` (string, unique)
* `Name`
* `Description`
* `ProductCode`
* `IsActive`
* `Priority` (int, for evaluation order)
* `CreatedAtUtc`
* `UpdatedAtUtc`
* `CreatedBy`
* `UpdatedBy`

---

### Example

```text
SYNQ_FUND.approval.limit
SYNQ_CARECONNECT.referral.region
```

---

## 2. Policy Rule Definition

### Create Entity

`PolicyRule`

Fields:

* `Id`
* `PolicyId`
* `ConditionType` (enum/string)
* `Field` (e.g., "amount", "organizationId")
* `Operator` (e.g., `==`, `>`, `<`, `IN`)
* `Value` (string/JSON)
* `LogicalGroup` (AND/OR grouping)
* `CreatedAtUtc`

---

### Examples

```text
amount < 50000
organizationId == user.organizationId
region IN ["CA", "NV"]
```

---

## 3. Policy → Permission Mapping

### Create Entity

`PermissionPolicy`

Fields:

* `Id`
* `PermissionCode`
* `PolicyId`
* `IsActive`

---

### Purpose

Defines:

```text
Permission → Conditions
```

Example:

```text
SYNQ_FUND.application:approve
→ Policy: amount < 50000
```

## 5. Policy Evaluation Engine

### Create Service

```csharp
IPolicyEvaluationService
```

---

### Method

```csharp
EvaluateAsync(user, permission, resourceContext)
```

---

### Behavior

1. check permission exists
2. retrieve linked policies
3. evaluate rules
4. combine results (AND/OR)
5. return:

```json
{
  "allowed": true/false,
  "reason": "...",
  "matchedPolicies": [...]
}
```

---

## 6. Enforcement Integration

### Update Filter

Enhance:

```csharp
RequirePermissionFilter
```

---

### Flow

```text
Check permission claim
→ If no policy → allow
→ If policy exists → evaluate
→ If pass → allow
→ If fail → deny
```

---

### Optional Config

```text
EnablePolicyEvaluation = true/false
```
