# LS-TENANT-002 — Tenant User Management (User + Access + Roles + Groups)

## Objective

Implement the **Tenant User Management UI** enabling tenant admins to:

* view all users in their tenant
* manage direct product access
* manage direct role assignments
* manage group memberships
* view effective access and permission sources

This is the **first fully functional tenant authorization feature**, built on top of LS-COR-AUT.

At completion, tenant admins must be able to:

```text
View → Assign → Revoke → Understand → Verify
```

for user-level access.

## Architectural Principles

### Backend is Source of Truth

* all access data must come from LS-COR-AUT APIs
* no frontend-derived authorization logic

### Explainability First

* every assignment must show **where it comes from**

### Safe Mutations

* all changes must:

  * be validated
  * show confirmation
  * handle failure gracefully

## A. Identity Summary

## B. Direct Product Access

### Display

List of products:

```text
SYNQ_FUND
SYNQ_LIEN
SYNQ_CARECONNECT
```

---

### Actions

* Add product access
* Remove product access

---

### API

```text
POST /identity/api/admin/users/{id}/products
DELETE /identity/api/admin/users/{id}/products/{productCode}
```

---

## C. Direct Roles

### Display

List roles:

```text
SYNQ_FUND.FUNDER
SYNQ_LIEN.ADMIN
```

---

### Actions

* Assign role
* Remove role

---

### API

```text
POST /identity/api/admin/users/{id}/roles
DELETE /identity/api/admin/users/{id}/roles/{roleCode}
```

---

## D. Group Membership

### Display

List groups:

* Group name
* Role source

---

### Actions

* Add to group
* Remove from group

---

### API

```text
POST /identity/api/admin/groups/{groupId}/members
DELETE /identity/api/admin/groups/{groupId}/members/{userId}
```

---

## E. Effective Access (CRITICAL)

### Display

#### Products

* full list of effective products

#### Roles

* role list with source:

  * Direct
  * Group (Group Name)

#### Permissions

* permission codes:

```text

```

### Data Source
