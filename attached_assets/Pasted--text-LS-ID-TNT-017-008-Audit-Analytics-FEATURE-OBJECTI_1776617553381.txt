# LS-ID-TNT-017-008 — Audit Analytics

## FEATURE OBJECTIVE

Implement an **Audit Analytics** layer that turns the existing audit and correlation system into an operational monitoring and insight capability.

This feature should enable the platform to aggregate and summarize audit data into useful analytical views such as:
- event volume trends
- access denial trends
- permission-change trends
- governance activity trends
- actor activity summaries
- tenant-scoped audit activity summaries
- basic anomaly-oriented operational signals (counts, spikes, top actors/resources)

This is an **analytics and aggregation feature**, not a full anomaly-detection engine and not a replacement for the raw audit viewer.

---

## WHY THIS FEATURE COMES NEXT

Current state:
- audit capture is complete
- permission-change audit is complete
- denied access audit is complete
- selective successful access audit is complete
- governance mutation audit is complete
- audit viewer is complete
- audit correlation engine is complete

What is missing:
- operators can investigate individual events and related chains
- but they still cannot easily answer:
  - Are denials increasing?
  - Which tenants/users are generating the most audit activity?
  - Which event categories are trending up?
  - Are there unusual spikes in audit exports, legal holds, or access denials?
  - Which products or event families are the most active?

This feature converts the audit system from:
- **forensic / investigative**
into:
- **operational / analytical**

---

DO NOT IMPLEMENT:
- machine-learning anomaly detection
- SIEM integrations
- external BI exports
- full custom dashboard builder
- broad data warehouse redesign
- graph analytics
- predictive analytics
- speculative metrics not grounded in existing audit fields
- per-event viewer redesign
- alerting/notification engine (that is a future feature)

ONLY IMPLEMENT:
- analytics query and aggregation support over the existing audit event store
- a practical analytics UI in Control Center
- trend, count, grouping, and summary views based on real event fields
- viewer-ready charts/tables/cards for audit analytics
- report-first implementation and honest documentation of supported analytics
