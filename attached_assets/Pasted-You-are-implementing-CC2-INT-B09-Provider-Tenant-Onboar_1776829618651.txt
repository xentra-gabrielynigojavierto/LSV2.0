OBJECTIVE

Enable providers (currently in COMMON_PORTAL stage) to:

- create their own tenant account OR join an existing tenant
- transition to TENANT stage
- access Synq CareConnect through their tenant portal
- retain continuity of identity and provider record (NO duplication)

At the end of this block:

- provider can onboard into a tenant
- provider AccessStage transitions to TENANT
- provider can log into tenant portal
- CareConnect is available inside tenant portal for provider

- provider.AccessStage == COMMON_PORTAL

PART B — BACKEND SERVICE

--------------------------------------------------

B.1 Create service

ProviderOnboardingService (new or existing service layer)

Method:

ProvisionProviderToTenantAsync:

Input:
- providerId
- tenantName
- tenantCode (subdomain)
- optional: join token (future)

--------------------------------------------------

B.2 Logic

1. Load provider from shared registry
2. Validate:
   - provider exists
   - provider.AccessStage >= COMMON_PORTAL

3. Create tenant (if new):
   - call Identity service
   - create organization / tenant
   - assign subdomain

4. Link provider to tenant:
   - update provider.TenantId = new tenantId

5. Call:
   provider.MarkTenantProvisioned(tenantId)

6. Persist changes

7. Assign roles in Identity:
   - user becomes tenant admin or provider role

--------------------------------------------------
