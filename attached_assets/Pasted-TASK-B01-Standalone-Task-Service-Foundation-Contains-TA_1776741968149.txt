TASK-B01 — Standalone Task Service Foundation
(Contains TASK-001, TASK-002, TASK-003)

OBJECTIVE:
Build the foundational standalone Task service (`task`) with core domain, persistence, CRUD APIs, notes, and observability. This must be platform-ready and NOT tied to any product (Liens, CareConnect, etc.).

NON-NEGOTIABLE RULES:
- Service name: task
- Database: tasks_db
- MUST be standalone and reusable
- NO product-specific assumptions
- MUST support:
  - product-scoped tasks
  - general cross-product task list
- MUST create report FIRST and update incrementally
- MUST follow existing LegalSynq service patterns
