## STEP 2 — DESIGN BACKEND LAST ADMIN PROTECTION

Design server-side enforcement for the last active tenant admin using authoritative backend data.

Requirements:
- Determine tenant from authenticated caller and/or target user
- Determine whether the target user currently holds TenantAdmin
- Count active TenantAdmin users for that same tenant using backend data
- Block actions that would leave the tenant with zero active TenantAdmin users

Minimum protected paths:
1. Deactivate last active TenantAdmin
2. Revoke TenantAdmin from last active TenantAdmin
3. Any replacement/downgrade flow that removes the final active TenantAdmin path if such flow exists

Requirements:
- Protection must apply for:
  - tenant portal UI calls
  - platform admin tools
  - direct API calls
  - self-targeted actions
- Protection must be authoritative and not depend on frontend-provided role state
