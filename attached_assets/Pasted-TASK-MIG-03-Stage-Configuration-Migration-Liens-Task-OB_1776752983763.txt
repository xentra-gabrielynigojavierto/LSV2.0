TASK-MIG-03 — Stage Configuration Migration (Liens → Task)

OBJECTIVE:
Migrate Liens task stage configuration from `liens_WorkflowConfigs` and `liens_WorkflowStages` into the Task service (`tasks_StageConfigs`) so Task becomes the primary owner of product-scoped task-board stage definitions for Liens.

This step migrates **stage definitions only**.

NON-NEGOTIABLE RULES:
- DO NOT migrate Flow orchestration stages
- DO NOT migrate orchestration transitions
- DO NOT redesign workflow logic
- DO NOT delete Liens stage tables in this step
- DO NOT migrate stage-transition rules in this prompt
- ZERO UI behavior regression
- MUST preserve Liens task-board behavior
- MUST support rollback
- MUST create report FIRST and update incrementally
- MUST inspect actual code/schema before implementation
- NO placeholders

CRITICAL BOUNDARY:
- `liens_WorkflowStages` = task-board / kanban stage config → belongs in Task
- `flow_workflow_stages` = orchestration step graph → stays in Flow
- Do NOT blur these concepts anywhere in implementation
