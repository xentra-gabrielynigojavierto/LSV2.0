TASK-MIG-09 — Final Cleanup (Remove Liens Config Authority)

OBJECTIVE:
Complete the Liens → Task configuration migration by removing Liens as a configuration system of record for task governance and templates, eliminating fallback and mirror logic, and preparing the legacy Liens config tables for safe removal.

This is the FINAL cleanup step of the migration lane.

NON-NEGOTIABLE RULES:
- ZERO UI behavior regression
- ZERO change to task generation, governance enforcement, or task-board behavior
- MUST preserve all current runtime behavior from the user’s perspective
- MUST create report FIRST and update incrementally
- MUST validate against actual code/schema before implementation
- MUST support rollback planning even though this step removes safety nets
- DO NOT touch Flow orchestration tables in this prompt
- DO NOT redesign generation rules in this prompt
- DO NOT introduce new features
- NO placeholders

CRITICAL GOAL:
After this step:
- Liens no longer owns task templates or governance config
- Liens no longer reads fallback config from its own DB for templates/governance
- Liens no longer mirrors template/governance writes to its DB
- legacy Liens config tables for templates/governance are removed or fully disconnected
- Task is the only authoritative source for these config domains

IMPORTANT SCOPE BOUNDARY:
This prompt is primarily for:
- `liens_TaskTemplates`
- `liens_TaskGovernanceSettings`
- related Liens repositories/services/sync/fallback logic

Do NOT remove stage/transition tables unless code inspection proves they are fully migrated and no longer authoritative. If they are not ready, document them as remaining follow-up items and leave them alone.
