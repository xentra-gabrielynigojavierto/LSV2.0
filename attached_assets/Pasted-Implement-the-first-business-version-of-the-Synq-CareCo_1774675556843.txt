Implement the first business version of the Synq CareConnect service for the LegalSynq .NET monorepo.

Context:
- Gateway is already working
- Identity is already working
- JWT issuance and validation are working
- MySQL on AWS RDS is already being used
- Monorepo structure already exists
- I want to pause Fund for now and build CareConnect first

Create CareConnect as a protected business service behind Gateway using:
- .NET 8
- EF Core
- MySQL
- tenant-scoped CRUD
- JWT claim-based tenant filtering
- clean architecture

- Provider

5. Add EF Core DbContext
Create CareConnectDbContext in CareConnect.Infrastructure.
Use Fluent API only and IEntityTypeConfiguration classes.

Table names:
- Providers
- Categories
- ProviderCategories
- Referrals

Primary keys:
- Providers: Id
- Categories: Id
- Referrals: Id
- ProviderCategories: composite key (ProviderId, CategoryId)

Indexes:
- Provider unique on (TenantId, Email)
- Provider index on (TenantId, Name)
- Category unique on Code
- Referral index on (TenantId, Status)
- Referral index on (TenantId, ProviderId)
- Referral index on (TenantId, CreatedAtUtc)

- Email required, valid format, max 320
- Phone required, max 50
- AddressLine1 required
- City required
- State required
- PostalCode required

Referral validation:
- ProviderId required
- ClientFirstName required
- ClientLastName required
- ClientPhone required
- ClientEmail required, valid format
- RequestedService required
- Urgency must be one of:
  - Low
  - Normal
  - Urgent
  - Emergency
- Status on create should default to New

Referral status values:
- New
- Received
- Contacted
- Scheduled
- Completed
- Cancelled
