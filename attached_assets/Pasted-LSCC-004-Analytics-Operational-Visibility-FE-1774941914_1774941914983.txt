This implementation should make the dashboard useful for decision-making, not just workflow entry.

==================================================
METRIC DEFINITIONS (SOURCE OF TRUTH)
==================================================

Use these definitions exactly unless existing backend data model requires a well-documented adaptation.

--------------------------------------------------
A. REFERRAL FUNNEL METRICS
--------------------------------------------------

For a selected date range:

- Total Referrals = referrals created in range
- Accepted = referrals with status = Accepted
- Declined = referrals with status = Declined
- Scheduled = referrals with status = Scheduled
- Completed = referrals with status = Completed

Derived metrics:
- Acceptance Rate = Accepted / Total Referrals
- Scheduling Rate = Scheduled / Accepted
- Completion Rate = Completed / Scheduled

Rules:
- If denominator is 0, show 0%, not NaN or blank
- Use clear labels and tooltips or helper text where helpful

--------------------------------------------------
B. TIME-BASED METRICS
--------------------------------------------------

If timestamps/history are available, compute:

- Avg Time to Accept = AcceptedAt - CreatedAt
- Avg Time to Schedule = ScheduledAt - CreatedAt
- Avg Time to Complete = CompletedAt - CreatedAt

If exact timestamps for a status transition are not directly available:
- derive from history where practical
- if unavailable, omit the metric or show "—"
- document the limitation in the report

Do NOT invent timestamps.

--------------------------------------------------
C. APPOINTMENT METRICS
--------------------------------------------------

For a selected date range:

- Total Appointments
- Completed Appointments
- Cancelled Appointments
- NoShow Appointments

Derived:
- Appointment Completion Rate = Completed / Total Appointments
- NoShow Rate = NoShow / Total Appointments

Same zero-denominator rule applies.

--------------------------------------------------
D. PROVIDER PERFORMANCE (BASIC)
--------------------------------------------------

For providers visible to the current user scope:

- Referrals Received
- Acceptance Rate
- Avg Time to Accept
- Appointments Completed

Rules:
- Keep this basic
- Avoid overloading the UI with too many columns
- If a metric cannot be computed reliably, show "—"

==================================================
ROLE-BASED VISIBILITY
==================================================

Referrer:
- sees only their org-scoped analytics

Receiver:
- sees only received/inbound analytics

TenantAdmin:
- sees tenant-wide analytics, consistent with current dashboard/access conventions

Do NOT widen visibility beyond what backend access already allows.
