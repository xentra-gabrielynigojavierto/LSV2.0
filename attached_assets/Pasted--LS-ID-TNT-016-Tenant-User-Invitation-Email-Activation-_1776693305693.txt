# LS-ID-TNT-016 — Tenant User Invitation Email + Activation Flow

## FEATURE OBJECTIVE

Implement a complete, production-ready **tenant user invitation and activation flow** that connects the Tenant Portal invite action to the running Notification service and allows invited users to securely activate their account.

This feature must complete the user onboarding lifecycle by adding:

- invitation token generation
- invite email dispatch through Notification service
- secure invite acceptance flow
- password setup / account activation
- invite status handling
- audit events for invite lifecycle

This is the missing execution layer between:
- tenant user management
- notification delivery
- first-time user activation

## STEP 4 — DEFINE THE INVITE / ACTIVATION LIFECYCLE

Implement a clear lifecycle with deterministic states.

Minimum lifecycle:
- Invited
- Active
- Disabled

Behavior rules:
1. Admin invites new user
   - user record created or updated in Invited state
   - invite token generated
   - expiration set
   - invite email sent through Notification service
2. User clicks invite link
   - token validated
   - expiry checked
   - invited user loaded
3. User sets password / completes activation
   - account becomes Active
   - token invalidated
   - acceptance timestamp stored
4. Re-invite existing invited user
   - regenerate token
   - refresh expiry
   - resend email
   - do NOT create duplicate user
5. Inviting already active user
   - either block with clear message or support resend-onboarding only if current behavior warrants it
   - document the chosen rule
