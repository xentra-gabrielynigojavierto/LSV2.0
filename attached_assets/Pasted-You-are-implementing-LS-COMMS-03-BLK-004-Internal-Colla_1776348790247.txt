STEP 2 — DESIGN MENTION MODEL
--------------------------------------------------

Add a new entity:

### MessageMention

Fields:
- id
- tenantId
- messageId
- conversationId
- mentionedUserId
- mentionedByUserId
- createdAtUtc

Rules:
- Only internal users can be mentioned
- Mentions are always tied to a message
- Mentions must be tenant-scoped

---

Mention syntax:
- `@{userId}` (recommended for backend reliability)
OR
- `@username` (if user lookup already exists)

👉 Prefer ID-based parsing if possible

STEP 3 — IMPLEMENT MENTION PARSING
--------------------------------------------------

In MessageService (or a helper):

- Extract mentions from message body
- Validate:
  - user exists
  - user belongs to tenant
- Deduplicate mentions
- Ignore invalid mentions (do not fail message creation)

STEP 4 — PERSIST MENTIONS
--------------------------------------------------

On message creation:

- Store MessageMention records
- Link to:
  - messageId
  - conversationId

Ensure:
- idempotent behavior (no duplicates per message/user)
- transactional consistency with message creation

STEP 6 — NOTIFICATION INTEGRATION
--------------------------------------------------

Requirements:
- idempotent (no duplicate notifications)
- non-blocking (failure does not block message creation)
