# REQUIRED IMPLEMENTATION

## A. Email Delivery Integration (Notifications Service)

Replace mock email adapter with real integration.

### Requirements
- integrate with Notifications service (v2 shared service)
- send:
  - recipients
  - subject
  - message
  - attachment (exported file)

### Implementation
- update `EmailReportDeliveryAdapter`
- use HTTP client or shared client pattern
- return structured `DeliveryResult`

---

## B. SFTP Delivery Integration

Replace stub SFTP adapter.

### Requirements
- use SSH.NET or equivalent
- support:
  - host
  - port
  - username/password or key auth
  - remote path
- upload file

### Security
- credentials via config only
- no hardcoding

---

## C. File Storage Integration

Add persistent storage (S3 preferred).

### Requirements
- create storage adapter (e.g., `IFileStorageAdapter`)
- upload exported files
- return storage key / URL

### Update behavior
- export service optionally stores file
- return metadata instead of raw bytes when enabled

### Config
- Storage:Enabled
- Storage:Provider
- Storage:BucketName
- Storage:BasePath

---

## D. Real Data Adapter

Replace mock query adapter with real product data integration.

### Requirements
- implement at least one adapter (e.g., SynqLien)
- map report definition → real query
- return TabularResultSet

### Rules
- keep adapter isolated
- no direct DB access in service layer
