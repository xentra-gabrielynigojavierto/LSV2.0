# LS-TENANT-003 — Group Management (Tenant Scope)

## Objective

Implement **full tenant group management UI and workflows**, enabling tenant admins to:

* create and manage groups
* assign users to groups
* assign product access to groups
* assign roles to groups
* understand how groups drive inherited access

This feature unlocks **scalable access management via inheritance**, making LS-TENANT practical for real-world usage.

At completion, tenant admins must be able to:

```text
Create Group → Assign Users → Assign Roles/Products → Drive Access at Scale
```

## Architectural Principles

### Groups as Inheritance Engine

* groups are the primary mechanism for scalable access
* roles + products assigned to groups propagate to users

### Backend-Driven Truth

* all group data and mutations must use LS-COR-AUT APIs
* no client-side inference of access

### Explainability

* UI must clearly show how group assignments affect user access

## E. Effective Access Preview (NEW & IMPORTANT)

### Purpose

Show what access this group **grants to its members**

---

### Display

#### Products

* effective products from group

#### Roles

* roles assigned to group

#### Permissions

* permissions derived from roles

---

### Data Source

Preferred:

```text
GET /identity/api/tenants/{tenantId}/groups/{groupId}/access-preview
```

Fallback:

* derive from roles + permission mappings (backend recommended)

---

### Example

```text
Group: Finance Team

Grants:
- SYNQ_FUND
- Role: FUND_APPROVER

Permissions:
- SYNQ_FUND.application:approve
- SYNQ_FUND.application:view
```
