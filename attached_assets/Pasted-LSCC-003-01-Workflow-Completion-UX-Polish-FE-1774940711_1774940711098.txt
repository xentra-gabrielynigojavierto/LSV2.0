REQUIREMENTS
==================================================

--------------------------------------------------
A. REFERRAL TIMELINE
--------------------------------------------------

Route:
- /careconnect/referrals/{id}

Add component:
- ReferralTimeline

Behavior:
- Call:
  GET /api/referrals/{id}/history

Display:
- list of events:
  - status changes
  - timestamps
  - notes (if available)
  - actor (if available)

Requirements:
- reverse chronological order
- visually distinguish event types
- reuse styling from appointment timeline if available

Fallback:
- no history → show "No activity yet"

--------------------------------------------------
B. APPOINTMENT CONFIRM / NOSHOW
--------------------------------------------------

Route:
- /careconnect/appointments/{id}

Add actions (provider-only):

- Confirm → status = Confirmed
- Mark NoShow → status = NoShow

API:
- PUT /api/appointments/{id}

Requirements:
- visible only to provider role
- immediate UI update after action

Fallback:
- API failure → revert UI state + show error

--------------------------------------------------
C. APPOINTMENT RESCHEDULE
--------------------------------------------------

Route:
- /careconnect/appointments/{id}

Add action:
- Reschedule

Flow:
1. Click "Reschedule"
2. Open modal
3. Fetch availability:
   GET /api/providers/{id}/availability
4. Select new slot
5. Confirm

API:
- POST /api/appointments/{id}/reschedule

Requirements:
- reuse SlotPicker or booking components
- maintain referral context
- update UI after success

Fallback:
- slot conflict → refetch
- no slots → empty state
