# STEP 3 — INPUT VALIDATION HARDENING

Ensure all externally reachable endpoints enforce:

### Ticket endpoints
- page ≥ 1
- pageSize capped (e.g., max 100)
- id must be valid GUID
- search fields sanitized (length limits)

### Comment endpoint
- body required
- body max length enforced (align with backend validator)
- reject empty/whitespace-only payload

### General
- reject invalid enum values cleanly
- reject malformed GUIDs
- reject unexpected payload fields if strict validation pattern exists

Do NOT rely solely on frontend validation.

---

# STEP 4 — RATE LIMITING / ABUSE PROTECTION

Introduce rate limiting for:

### Customer endpoints ONLY (critical)
- /support/api/customer/*

Recommended policy:
- 60 requests per minute per user (JWT subject)
- burst allowed but bounded

If infrastructure supports:
- Use ASP.NET rate limiting middleware (fixed window or sliding window)
- Key by:
  - external_customer_id (preferred)
  - fallback: user identifier / IP

Do NOT apply aggressive rate limiting to internal/admin endpoints.

---

# STEP 5 — PAYLOAD SIZE LIMITS

Enforce:
- Max request body size for comment endpoint (e.g., 8KB or align with validator)
- Max query length limits

Use:
- ASP.NET request size limits or middleware

# STEP 6 — ERROR HANDLING STANDARDIZATION

Ensure all endpoints return safe, consistent responses:

### Requirements
- No stack traces
- No internal exception messages
- No SQL errors exposed

### Standard format (if not already implemented):
{
  "error": {
    "code": "string",
    "message": "safe user-facing message",
    "correlationId": "optional"
  }
}

### Behavior:
- 400 → validation errors
- 401 → auth required
- 403 → access denied / feature disabled
- 404 → not found (do not leak ownership)
- 500 → generic error

Ensure:
- customer endpoints do not leak whether a ticket exists outside their scope

---

# STEP 7 — SECURITY HEADERS

Ensure API responses include (if not already handled globally):

- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY
- X-XSS-Protection: 0 or appropriate modern config
- Strict-Transport-Security (if HTTPS enforced)
- Content-Security-Policy (if applicable at gateway level)

If headers are already enforced at gateway, document instead of duplicating.
