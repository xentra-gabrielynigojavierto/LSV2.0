You are working inside the LegalSynq v2 codebase.

Objective:
Integrate the Liens service with the v2 Documents service so that a finalized sale generates and attaches a BillOfSale PDF, storing the returned DocumentId on the BillOfSale record.

This feature introduces the first real Liens → Documents integration and must preserve clean service boundaries.

NON-NEGOTIABLE ARCHITECTURE RULE:
All document storage MUST be handled by the v2 Documents service.
- Liens must NOT store files
- Liens must NOT create document tables
- Liens must ONLY store DocumentId references

CRITICAL IMPLEMENTATION RULES:
- Do NOT break the existing sale finalization workflow
- Do NOT duplicate business logic from `LienSaleService`
- Keep PDF generation isolated and replaceable
- Keep Documents integration behind an abstraction
- Do NOT introduce cross-service DB joins
- Do NOT expose Documents internals to API layer
