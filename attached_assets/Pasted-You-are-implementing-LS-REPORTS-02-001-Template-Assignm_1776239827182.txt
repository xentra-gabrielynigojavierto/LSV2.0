# STORY ID
**LS-REPORTS-02-001 — Template Assignment & Tenant Customization Foundation**

---

# OBJECTIVE
Introduce the first tenant-aware template distribution layer for the Reports Service so `ReportTemplate` records can be:
- assigned globally
- assigned to specific tenants
- resolved by tenant, product, and organization type
- prepared for tenant-level customization and commercialization in later stories

This story must establish the foundation for:
- control-center-managed template distribution
- tenant-specific availability
- product/org-type filtering
- future pricing-tier and feature-gated reporting

## F. Assignment Rules
Implement at minimum:

### Rule 1 — Scope validity
- `Global` assignment must not require tenant list
- `Tenant` assignment must include at least one tenant

### Rule 2 — Product/org alignment
- assignment product/org type must align with the template’s product/org type
- reject conflicting assignment requests

### Rule 3 — Published version requirement
Tenant-visible catalog should only return templates where:
- template is active
- assignment is active
- a published version exists

### Rule 4 — Duplicate handling
Prevent invalid duplicates such as:
- the same template assigned globally more than once as active
- the same tenant assigned redundantly in a conflicting active assignment

### Rule 5 — Activation model
Assignments and assignment-tenant rows should support activation/deactivation without requiring physical deletion

## J. Validation and Error Handling
Implement at minimum:

### Assignment create

- scope valid

- tenant list empty/ignored for global scope

### Assignment update

- updated assignment must remain valid

### Tenant catalog query
- `TenantId` required
- `ProductCode` required
- `OrganizationType` required

### Status code guidance
- `400` invalid request
- `404` template/assignment not found
- `409` duplicate/conflict

Use lightweight consistent responses.
