**LS-REPORTS-01-001-01 — Persistence Model Alignment**

# OBJECTIVE
Align the persistence model with the correct domain language by refactoring:

- `ReportDefinition` → `ReportTemplate`

and updating related code artifacts so the service uses consistent product terminology before API and template management features are added.

---

# REQUIRED IMPLEMENTATION

## A. Rename Domain Entity
Rename:

- `ReportDefinition.cs` → `ReportTemplate.cs`
- `class ReportDefinition` → `class ReportTemplate`

Update all references across the solution.

---

## B. Update Related Entities

### `ReportExecution`
Update:
- `ReportDefinitionId` → `ReportTemplateId`
- navigation property from `ReportDefinition` to `ReportTemplate`

### `ReportTemplateVersion`
Update:
- FK/reference from `ReportDefinitionId` to `ReportTemplateId`
- navigation property from `ReportDefinition` to `ReportTemplate`

Do not change behavior — only align naming.

---

## C. Update DbContext
Update:

- `DbSet<ReportDefinition>` → `DbSet<ReportTemplate>`

Ensure all references compile and mappings still work.

---

## D. Update EF Configurations
Rename/update:

- `ReportDefinitionConfiguration.cs` → `ReportTemplateConfiguration.cs`

Update the configuration class name and any references accordingly.

### Important:
Prefer to keep the existing physical table name stable for now unless renaming is demonstrably safe and necessary.

If the current table is:
- `rpt_ReportDefinitions`

it is acceptable to keep that table name for now while aligning code names.
Document the decision clearly in the report.

---

## E. Update Repository Layer
Update repository interfaces and implementations so they use the new `ReportTemplate` naming consistently.

This includes:
- repository method return types
- parameters
- includes/navigation
- query code
- mock implementations

If `ITemplateRepository` already exists, align its method signatures to `ReportTemplate` terminology.

## F. Update Any Remaining References
Ensure there are no stale `ReportDefinition` references left in:
- Contracts
- Infrastructure
- Api
- Worker
- Shared
- tests if applicable

If any legacy reference must remain temporarily for migration reasons, document it clearly.
