TASK-FLOW-02 — Read Path Migration (Flow → Task)

OBJECTIVE:
Move Flow read paths off `flow_workflow_tasks` and onto the Task service so Flow no longer depends on its shadow task table for user task lists, task detail, queue eligibility reads, or SLA-driven status reads. This is Phase 2 of Flow → Task delegation and is the prerequisite to dropping `flow_workflow_tasks`.

CONTEXT:
Phase 1 already moved Flow task writes to Task service while retaining `flow_workflow_tasks` as a shadow read model. This prompt completes the read-side migration by:
- extending Task service query/read support where needed
- moving Flow read services to Task
- retiring Flow’s task-SLA background ownership
- preparing the system for final table drop in TASK-FLOW-03

IMPORTANT CONSTRAINTS DISCOVERED:
- Current Flow reads still depend on `flow_workflow_tasks` through `MyTasksService` and `WorkflowTaskSlaEvaluator`. :contentReference[oaicite:0]{index=0}
- Task service currently lacks native Flow queue semantics for `RoleQueue` / `OrgQueue`, so Phase 2 must add Task-side read support or a compatible representation before reads can move fully. :contentReference[oaicite:1]{index=1}
- Flow orchestration tables remain in `flow_db` and MUST NOT be moved in this prompt. 

NON-NEGOTIABLE RULES:
- DO NOT move Flow orchestration tables
- DO NOT merge Task and Flow databases
- DO NOT drop `flow_workflow_tasks` in this prompt
- DO NOT redesign the workflow engine
- DO NOT break My Tasks / queue behavior
- DO NOT remove rollback safety until validation is complete
- MUST create the report FIRST and update it incrementally
- MUST validate against the current codebase before implementing
- MUST preserve tenant isolation and current auth behavior
- NO placeholders

CRITICAL GOAL:
After this step:
- Flow reads task data from Task service, not `flow_workflow_tasks`
- Task service exposes enough query capability to support Flow read scenarios
- Flow’s task shadow table is no longer needed for normal runtime reads
- `flow_workflow_tasks` can be safely deprecated ahead of TASK-FLOW-03
