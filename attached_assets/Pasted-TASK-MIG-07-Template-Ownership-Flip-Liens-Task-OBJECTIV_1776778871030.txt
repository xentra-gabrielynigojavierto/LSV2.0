TASK-MIG-07 — Template Ownership Flip (Liens → Task)

OBJECTIVE:
Flip template ownership so the Task service becomes the authoritative write/read owner for Liens task templates, while Liens becomes a consumer of Task-hosted template data only. This step removes Liens template authority without changing user-facing behavior.

This is the FIRST ownership-flip step.

NON-NEGOTIABLE RULES:
- ZERO UI behavior regression
- ZERO change to template semantics
- MUST preserve generation behavior exactly
- MUST preserve existing template IDs
- MUST support rollback
- MUST create report FIRST and update incrementally
- MUST validate against actual code/schema before implementation
- DO NOT delete `liens_TaskTemplates` in this step
- DO NOT remove rollback-safe fallback immediately unless explicitly validated
- DO NOT migrate stage ownership again in this prompt
- NO placeholders

CRITICAL GOAL:
After this step:
- Task service is the primary write target for template admin operations
- Task service is the primary read source for runtime template use
- Liens DB template table is retained only as temporary rollback safety, not as authority
