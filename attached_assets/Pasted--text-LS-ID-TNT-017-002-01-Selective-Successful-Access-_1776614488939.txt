# LS-ID-TNT-017-002-01 — Selective Successful Access Audit

## FEATURE OBJECTIVE

Extend the audit system to capture **selective successful access events** for high-value protected resources without creating noisy, low-value, per-request audit spam.

This feature should answer:

- who successfully accessed a protected resource
- when they accessed it
- under which tenant/session
- what protected resource or operation was accessed
- under which authorization path it was allowed

This is a **targeted success-audit feature**, not blanket request logging.
