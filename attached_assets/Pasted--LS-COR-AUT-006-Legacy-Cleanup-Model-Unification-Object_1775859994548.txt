## Objective

Consolidate LS-COR-AUT into a **single, unified access control model** by removing legacy role/group systems and aligning all authorization behavior to:

* LS-COR-AUT-002 (source-of-truth)
* LS-COR-AUT-003 (effective access + JWT)
* LS-COR-AUT-004 (groups + inheritance)

This feature eliminates:

* duplicate role systems
* legacy group APIs
* inconsistent claim formats

At completion:

* there must be **one role system**
* **one group system**
* **one claim format**
* **one access resolution path**

## In Scope

### 1. Remove Legacy Role System

#### Remove:

* `ScopedRoleAssignment`
* `ProductRoleResolutionService`
* any legacy role mapping logic used in LS-COR-AUT-001

#### Replace with:

* `UserRoleAssignment`
* `GroupRoleAssignment`
* `IEffectiveAccessService` (single source of truth)

---

### 2. Remove Legacy Group System

#### Remove:

* `/identity/api/admin/groups`
* legacy group entities and services
* any UI fallback paths using legacy groups

#### Ensure:

* all UI uses `/api/tenants/{tenantId}/groups`
* no references to legacy group endpoints remain

---

### 3. Unify JWT Claim Format

#### Current (temporary dual format):

```text
PRODUCT:Role
RoleCode
```

#### Final required format:

```text
product_roles:
[
  "SYNQ_CARECONNECT:CareCoordinator",
  "SYNQ_FUND:FundManager"
]
```

#### Actions:

* Remove legacy role-only claims
* Remove merge logic
* Ensure all filters and services use product-scoped roles

---

### 4. Update LS-COR-AUT-001 Filters

Modify:

* `RequireProductAccessFilter`
* `RequireProductRoleFilter`

#### Ensure:

* filters read `product_roles` in unified format
* role checks include product context
* no dependency on legacy role resolution

---

### 5. Clean Effective Access Service

Ensure `IEffectiveAccessService` is:

* the ONLY source for:

  * products
  * roles
  * claim generation

Remove:

* any fallback logic
* any legacy merging

---

### 6. Remove Dual Data Dependencies

Ensure:

* no code references:

  * legacy role tables
  * legacy group tables

* no mixed queries combining:

  * new + old models

---

### 7. Data Migration / Cleanup

If legacy data exists:

* migrate active assignments to new tables
* or explicitly mark legacy data as deprecated

Ensure:

* no orphaned references
* no runtime dependency on legacy data

---

### 8. AccessVersion Integrity

Ensure:

* all mutations (direct + group) still trigger AccessVersion updates
* removal of legacy paths does NOT break versioning
